❯ 什么都不知道                                                                  

● 好，零基础反而走得最快。

  第二步：框架构建

  先给你一个最小框架，今天就能用。

  JavaScript 就干一件事：让网页会动。

  想象网页是一张纸——HTML 是纸上的字，CSS 是字的颜色大小排版。而 JavaScript
  是让纸上的东西能回应你：你点一下，它变一下。

  ---
  三个最最基本的概念：

  1. 变量 — 记住一个东西
  let 名字 = "张三"   // 记住"张三"这个名字

  2. 函数 — 做一件事
  function 说你好() {
    alert("你好！")   // 弹出"你好！"
  }

  3. 事件 — 有人做了什么，就自动做另一件事
  按钮.onclick = 说你好   // 点按钮，就弹出"你好！"

  ---
  问题：

  「这三个里，哪个让你觉得最奇怪、最想搞清楚？或者说，哪一句让你脑子里冒问号了？
  」

❯ 事件

  第三步：问题穿透

  先出一道题，想想看：

  ---
  题目：

  我写了一个网页，上面有一个红色按钮。我希望：点一下红色按钮，它就变成蓝色。

  你能猜到，这里面涉及到几个"角色"？比如谁、谁、谁？

  （不用写代码，不用准确，告诉我你的直觉想法就行）

  ---

❯ 我不清楚                                                                      

● 没关系，给你一个小线索：

  ▎ 不是让你想"代码怎么写"，而是想——这个场景里，有哪几个东西在"说话"？

  比如：你现在坐在电脑前，屏幕上是按钮。你点了一下按钮，然后按钮变色了。

  这中间发生了什么？

  - 谁发出了一个动作？
  - 谁感觉到这个动作？
  - 谁去做了变色这件事？

  试着把这三个人（或东西）找出来，不用代码，就用大白话说。

  ---
  想一想，再告诉我。

❯ 用户发出点击动作，按钮感觉到到了点击，后面不知道了                            

● 很好，两个都说对了！

  还差一个——谁来真正执行"变蓝色"这件事？

  不是按钮本身，按钮只负责"被点"，它自己不决定自己要变蓝。

  是另一个人，你来猜它是哪个？

  ---
  再给你一个线索：

  ▎ 变蓝色需要一个命令，这个命令从哪里来？是从按钮那里发给它的吗？
